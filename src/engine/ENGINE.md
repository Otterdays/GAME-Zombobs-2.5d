# ZOMBS Engine Documentation

## Overview
The **ZOMBS Engine** is a high-performance, WebGPU-based 2.5D game engine written in Rust and compiled to WebAssembly. It is designed for browser-based games that require rendering hundreds of entities without lag.

---

## Architecture

```
src/engine/
├── mod.rs          # Module re-exports
├── renderer.rs     # WebGPU Renderer (Instanced Cubes)
├── components.rs   # ECS Components (Position, Velocity, Renderable, etc.)
├── systems.rs      # ECS Systems (Movement, Collision, Projectiles, etc.)
├── input.rs        # Input State Management
├── ecs.rs          # ECS utilities (placeholder)
└── shader.wgsl     # GPU Shader (Procedural Cubes)
```

## Core Concepts

### 1. Isometric 2.5D Rendering
The engine uses an **orthographic camera** tilted at 45 degrees to create a classic isometric perspective. All game objects are rendered as instanced cubes with procedural shading.

**Key Parameters:**
- `view_height`: 10.0 world units (controls zoom level)
- `Camera.position`: Vec2 tracking the player
- `Camera.screen_to_world()`: Converts mouse screen coordinates to world coordinates via ray-plane intersection

### 2. Entity Component System (ECS)
We use the **hecs** crate for our ECS. Entities are just IDs; components are attached data.

**Core Components:**
| Component | Description |
|-----------|-------------|
| `Position` | 3D World Position (x, y, z) |
| `Velocity` | 2D Movement Vector (x, y) |
| `Renderable` | Visual Dimensions + Color |
| `AABB` | Collision Hitbox (width, height) |
| `Player` | Marker for player-controlled entity |
| `Projectile` | Time-to-live for bullets |
| `GlobalRotation` | Quaternion rotation for facing |
| `LocalTransform` | Offset from Parent entity |
| `Parent` | Hierarchy link to parent entity |
| `BodyPart` | Limb type for animation |

### 3. Systems
Systems run each frame and operate on component queries.

**Core Systems:**
| System | Purpose |
|--------|---------|
| `player_input_system` | Reads WASD + Mouse, sets velocity and rotation |
| `movement_system` | Applies velocity, handles AABB collision |
| `animation_system` | Animates limbs based on velocity |
| `transform_propagation_system` | Resolves parent-child hierarchies |
| `projectile_system` | Despawns expired bullets |

## Collision System

The collision system uses **Axis-Aligned Bounding Boxes (AABB)**.

**Algorithm:**
1. Collect all static colliders (entities with `AABB` but no `Velocity`).
2. For each dynamic entity (has `AABB` + `Velocity`):
   - Try moving on X axis. If collision detected, cancel X movement.
   - Try moving on Y axis. If collision detected, cancel Y movement.
3. This creates a **sliding** effect along walls.

## Procedural Graphics

All visuals are generated procedurally in the shader:
- Cubes are defined as 36 vertices (12 triangles) in `shader.wgsl`
- Face shading is calculated based on face normal (top is brightest)
- Ground plane uses noise-based grass/dirt pattern

**No texture files are used.** This keeps the WASM bundle tiny.

## Usage

```rust
// In game.rs
use crate::engine::renderer::{Renderer, Camera};
use crate::engine::components::{Position, Velocity, AABB, Player};
use crate::engine::systems::{movement_system, player_input_system};

// Spawn a player
let player = world.spawn((
    Position::new_3d(0.0, 0.0, 0.0),
    Velocity::new(0.0, 0.0),
    AABB::new(0.4, 0.4),
));
world.insert_one(player, Player { id: 1 }).unwrap();
```
